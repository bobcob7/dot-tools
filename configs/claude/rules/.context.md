# configs/claude/rules

## Purpose

Persistent rules that Claude Code follows across all conversations. Rules provide consistent behavior guidelines.

## Functionality

- Coding conventions and best practices
- Git workflow and commit standards
- Testing guidelines

## Files

| File | Description |
|------|-------------|
| `git-workflow.md` | Git commit conventions (imperative mood, conventional prefixes) and branch safety rules |
| `error-handling.md` | Error handling patterns (early returns, try-catch boundaries) and anti-patterns to avoid |
| `testing.md` | Test structure guidelines (naming, arrangement, mocking) and what to test |
