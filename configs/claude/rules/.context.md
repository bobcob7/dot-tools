# configs/claude/rules

## Purpose

Persistent rules that Claude Code follows across all conversations. Rules provide consistent behavior guidelines.

## Functionality

- Coding conventions and best practices
- Git workflow and commit standards
- Testing guidelines

## Files

| File | Description |
|------|-------------|
| `git-workflow.md` | Git commit conventions (imperative mood, conventional prefixes) and branch safety rules |
| `go-standards.md` | Go language standards: code style, interfaces, error handling, moq, testing, DI |
| `typescript-standards.md` | TypeScript/React standards: code style, types, components, Redux, Vitest testing |
| `error-handling.md` | Error handling patterns (early returns, try-catch boundaries, Go-specific) and anti-patterns |
| `testing.md` | Test structure guidelines (naming, mocking) with Go and frontend-specific sections |
